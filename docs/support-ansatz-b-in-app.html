Setup-Schritte

  • 1 support@korrekturpilot.de bei all-inkl anlegen (wie Ansatz A)
  • 2 Support-Formular als React-Komponente bauen (SupportWidget.tsx)
  • 3 API-Route /api/support erstellen: sendet E-Mail + optional Notion API
  • 4 Floating Action Button in Layout einbauen (auf allen eingeloggten Seiten)
  • 5 Notion Database "Support-Inbox" anlegen (Status, Kategorie, Prioritaet)
  • 6 Optional: Notion API Integration fuer automatisches Ticket-Erstellen

Bewertung

Vorteile

  • Strukturierte Tickets (Kategorie, Screenshot)
  • Kontext automatisch mitgeliefert (welche Klausur)
  • Ticket-Tracking in Notion (Status, Prioritaet)
  • Screenshot-Upload direkt in der App
  • Professioneller Eindruck bei Lehrkraeften
  • Skaliert bis 500+ Tickets

Nachteile

  • Entwicklungsaufwand: ~4-6 Stunden
  • Notion API Setup erforderlich
  • Neues Tool (Notion) fuer Team
  • Screenshot-Upload braucht Supabase Storage
